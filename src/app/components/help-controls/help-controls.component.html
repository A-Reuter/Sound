Controls

Display Configuration

Mode Selection

the display mode can be changed using the selector switch over the top right corner of the canvas

there are three modes available to choose from:

  • default  - highlights source and sink places, as well as enabled transitions
  • traveled - highlights parts of the net that are present within the active firing sequence or the sequence log
  • errors   - highlights parts of the net that are directly or indirectly tied to the invalidity of a net marking

Showing & Hiding Elements

individual display elements can be enabled or disabled through the options available under the Show / Hide Elements tab of the View menu

Color Legend

the canvas color legend can be accessed using the small info button located to the right of the display mode selector

Error Information

whenever an error occurs, a panel containing additional information can be accessed using the Question Mark button which appears in the top right corner of the canvas

after at least one error has occurred, a panel containing information about previously encountered errors can be accessed using the Warning button which appears in the same location

File Loading

General

petri nets may be loaded from .pnml, .json, .txt or .sav files

certain restrictions apply regarding individual formats:

  • pnml     - all files adhering to the 2009 version of the Petri Net Markup Language grammar should work without issue
  • json,txt - only supports certain proprietary formats generated and exported by various tools available on the ilovepetrinets page
  • sav      - used for exporting and reimporting savestates of this application, will not work with any other sav files

Drag & Drop

files can be loaded by dragging them onto the canvas, e.g. from the example files section or from another window

File Selection

to select a file via a standard selection window, use the Load File option within the File menu

Example Files

leftmost within the control bar above the canvas, three example files are available to load through drag & drop

every example showcases one of the three conditions of the soundness property not being met:

  • example A contains a workflow net that is not sound because the modelled process does not always terminate
  • example B contains a workflow net that is not sound because the modelled process does not always terminate correctly
  • example C contains a workflow net that is not sound because the modelled process contains an activity that can never be executed

Basic Canvas Interactions

moving a node

  • to move a node, hover over it, press and hold the primary (left) mouse button, and then drag it across the canvas

marking a node

  • to mark a node or arc, hover over it and press the tertiary (middle) mouse button or the M key (press again to unmark)
  • marking is a strictly visual feature which only serves to allow the emphasis of elements that are of particular interest or to improve orientation when switching between different display modes

displaying node information

  • a panel with detailed information about a specific node will be displayed when hovering over that node
  • this panel can be shown permanently by pressing the I key when targeting the node (press again to deactivate)

arranging the net

  • to toggle between free and automated net arrangement, press the Spring Embedder button located on the right within the control row above the canvas, or use the Toggle Spring Embedder option within the View menu
  • to try to force a net layout where all source places are located to the left while all sink places are located to the right, use the Force Net Layout option within the View menu

setting the view focus

  • to focus the canvas view on the currently displayed net, press the Focus button located on the right within the control row above the canvas, or use the Focus View option within the View menu

changing the view area

  • to increase or decrease the area visible on the canvas, use the mouse scroll wheel whilst hovering over the canvas

Sequence Control

firing a transition

  • to fire an enabled transition, hover over it and press the primary (left) mouse button

undoing a firing action

  • to roll back a previously executed firing action, press the Back button located centrally above the canvas, or use the Undo Step option within the Edit menu

redoing a firing action

  • to restore a previously undone firing action, press the Forward button located centrally above the canvas, or use the Redo Step option within the Edit menu

resetting the net marking

  • to reset the net marking, press the Reset button located centrally above the canvas, or use the Reset Net option within the Edit menu

loading a log sequence

  • to load a sequence from the log, hover over it and press the primary (left) mouse button
  • sequences which terminate successfully are framed in green, sequences leading to an invalid marking are framed in red

retracing a log sequence

  • sequences loaded from the log always start with the net in the initial state
  • to skip forward within a sequence loaded from the log, press the Forward button located centrally above the canvas, or use the Redo Step option within the Edit menu
  • to skip backward within a sequence loaded from the log, press the Back button located centrally above the canvas, or use the Undo Step option within the Edit menu

automating sequence execution

  • to start the automated execution of random firing sequences, press the Play button located centrally above the canvas, or use the Start Automation option within the Edit menu
  • the algorithm will generate sequences by repeatedly choosing a random transition from all enabled transitions and firing it
  • when a sequence terminates, the algorithm will reset the net and begin a new sequence, looping until it is manually stopped

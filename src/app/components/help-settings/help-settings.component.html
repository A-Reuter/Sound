Settings

Display

Modes

  • default  - highlights net elements most relevant when executing firing sequences: source places, sink places and enabled transitions
  • traveled - highlights all parts of the net previously traveled across: differentiates between elements present within completed steps of the active sequence, elements present within future steps of the active sequence ald elements present within the sequence log
  • errors   - highlights parts of the net that are directly or indirectly tied to the invalidity of a net marking

Triggered Mode Switching

  • on error     - while enabled, the display mode is automatically switched to the errors setting when an invalid marking is encountered
  • on file load - while enabled, the display mode is automatically switched to the default setting when a new file is loaded
  • on log load  - while enabled, the display mode is automatically switched to the traveled setting when a log sequence is loaded
  • on autorun   - while enabled, the display mode is automatically switched to the traveled setting when the automated firing sequence execution is enabled

Spring Embedder

Exempt Dragged Nodes

while enabled, the spring embedder applies no forces to a dragged node, resulting in an unimpeded drag

while disabled, the spring embedder applies forces to a dragged node in the same way forces are applied to all other nodes, sacrificing drag responsiveness for more consistent node arrangement

Node Tethering

controls the degree to which the nodes of a petri net are tethered together by the spring embedder algorithm

  • loose    - nodes are only loosely tethered together, allowing for a high degree of positioning flexibility
  • balanced - nodes are moderately tethered together, allowing for a mostly sufficient degree of positioning flexibility
  • tight    - nodes are more tightly tethered together, allowing only for a limited degree of positioning flexibility

Sequence Automation

Iteration Delay

controls the amount of time the automated firing sequence execution idles after each firing action before executing the next

(effectively equates to the frequency with which firing actions are executed)

Workflow Checks

Use Narrow Definition

applies a more narrow class definition when checking whether a given petri net is a workflow net: while enabled, in addition to the default set of checks directly pertaining to the formal workflow net definition, only arc weights up to 1 will be allowed

File Set

allows to switch the file set displayed left within the control bar above the canvas

  • examples  - displays a set of example files which are pre-analyzed for demonstration
  • exercises - displays a set of exercise files which are not pre-analyzed

Notifications

On Application Malfunction

controls the way the user is notified about the occurrence of any expected, yet unsolvable software malfunction

  • dialog - displays a dialog window
  • popup  - displays a popup window
  • toast  - displays a toast message

For Confirmation

controls the way the user is approached to confirm an action

  • dialog - displays a dialog window
  • popup  - displays a popup window
  • auto   - displays no window, automatically confirming to proceed with the action

For Information

controls the way the user is presented important information

  • dialog - displays a dialog window
  • popup  - displays a popup window
  • toast  - displays a toast message
  • none   - displays no info
